# Marathi NER

Utilities for preparing Marathi named entity recognition data and running
quick inference checks with GLiNER and L3Cube MahaBERT models.

The current implementation supports:

- downloading L3Cube Marathi NER sources from `l3cube-pune/MarathiNLP`
- converting L3Cube IOB TSV files to GLiNER JSONL format
- running a GLiNER Marathi smoke test
- running token-classification inference with an L3Cube MahaBERT NER model

## Repository Layout

```text
scripts/download_datasets.py       Download configured upstream datasets
scripts/convert_iob_to_gliner.py   Convert IOB TSV data to GLiNER JSONL
scripts/test_gliner_marathi.py     Run a quick GLiNER inference check
scripts/infer_mahabert_ner.py      Run MahaBERT token-classification inference
docs/marathi_ner_sources.md        Dataset source and redistribution notes
docs/gliner_conversion.md          Conversion format details
data/raw/source_manifest.json      Local source manifest generated by downloader
data/processed/gliner/             Local converted GLiNER JSONL outputs
```

## Setup

Use Python 3.10+.

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
```

Install dependencies only for the workflow you need:

```bash
# Dataset download and conversion use the Python standard library plus git.

# GLiNER smoke test
python3 -m pip install gliner

# MahaBERT / Transformers inference
python3 -m pip install transformers torch
```

## Download Datasets

```bash
python3 scripts/download_datasets.py
```

By default this downloads:

- `l3cube-mahaner`
- `l3cube-mahasocialner`

Raw upstream files are stored under `data/raw/upstream/`, and source metadata is
written to `data/raw/source_manifest.json`.

To download one source:

```bash
python3 scripts/download_datasets.py --source l3cube-mahaner
```

See `docs/marathi_ner_sources.md` for dataset source and redistribution notes.

## Convert To GLiNER

```bash
python3 scripts/convert_iob_to_gliner.py
```

Default outputs:

```text
data/processed/gliner/train.jsonl
data/processed/gliner/valid.jsonl
data/processed/gliner/test.jsonl
data/processed/gliner/metadata.json
```

Current local generated split sizes:

| Split | Examples |
| --- | ---: |
| train | 33,608 |
| valid | 3,000 |
| test | 4,249 |

The converter maps the L3Cube labels to:

| Raw tag | Label |
| --- | --- |
| `NEP` | `person` |
| `NEL` | `location` |
| `NEO` | `organization` |
| `NEM` | `measure` |
| `NETI` | `time` |
| `NED` | `date` |
| `ED` | `designation` |

Useful conversion options:

```bash
# Include source metadata in each example
python3 scripts/convert_iob_to_gliner.py --include-provenance

# Fail on malformed IOB continuation tags instead of repairing them
python3 scripts/convert_iob_to_gliner.py --strict-iob

# Convert a single split or source
python3 scripts/convert_iob_to_gliner.py --split train --source l3cube-mahaner
```

See `docs/gliner_conversion.md` for format details.

## Run Inference Checks

GLiNER:

```bash
python3 scripts/test_gliner_marathi.py
```

With custom labels:

```bash
python3 scripts/test_gliner_marathi.py \
  --labels person,organization,location,date \
  --threshold 0.15
```

MahaBERT token classification:

```bash
python3 scripts/infer_mahabert_ner.py
```

With a specific model and JSON output:

```bash
python3 scripts/infer_mahabert_ner.py \
  --model l3cube-pune/marathi-ner \
  --json
```

## Data Notes

The tracked source code can regenerate the dataset artifacts. Local generated
data under `data/raw/upstream/` and `data/processed/gliner/` may be large and
should be treated according to upstream dataset license and redistribution
terms.
