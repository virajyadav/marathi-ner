# Marathi NER

Create a cleaned Marathi named entity recognition dataset and train a lightweight
Marathi NER model.

The current implementation builds a GLiNER-compatible dataset from L3Cube
Marathi NER sources, exports data for Label Studio review, and includes a
notebook workflow for fine-tuning a compact GLiNER model.

## Objective

- Build a reusable Marathi NER dataset from public L3Cube sources.
- Normalize entity labels across source datasets.
- Remove sentences with no entity spans so training data contains useful NER
  examples only.
- Export train, validation, and test splits in GLiNER JSONL format.
- Support Label Studio import for manual inspection or correction.
- Fine-tune a lightweight Marathi NER model from a multilingual GLiNER base
  checkpoint.

## Setup

Use Python 3.10+.

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
```

Install only the dependencies needed for the workflow you are running:

```bash
# Dataset download and conversion use the Python standard library plus git.

# GLiNER inference and training
python3 -m pip install "gliner[training]"

# MahaBERT / Transformers inference
python3 -m pip install transformers torch
```

## Dataset Pipeline

### 1. Download Source Data

```bash
python3 scripts/download_datasets.py
```

By default this downloads:

- `l3cube-mahaner`
- `l3cube-mahasocialner`

Raw files are stored under:

```text
data/raw/upstream/l3cube-marathinlp/
```

Source metadata is written to:

```text
data/raw/source_manifest.json
```

To download one source:

```bash
python3 scripts/download_datasets.py --source l3cube-mahaner
```

### 2. Convert To GLiNER Format

```bash
python3 scripts/convert_iob_to_gliner.py
```

Default outputs:

```text
data/processed/gliner/train.jsonl
data/processed/gliner/valid.jsonl
data/processed/gliner/test.jsonl
data/processed/gliner/metadata.json
```

Each JSONL row uses tokenized GLiNER format:

```json
{"tokenized_text":["तेव्हापासून","रैना","त्याला"],"ner":[[1,1,"person"]]}
```

Sentences with no entities are skipped:

```json
{"tokenized_text":["..."],"ner":[]}
```

Current generated split sizes:

| Split | Examples | Tokens | Entities |
| --- | ---: | ---: | ---: |
| train | 22,749 | 442,575 | 57,607 |
| valid | 2,138 | 47,771 | 6,096 |
| test | 3,104 | 72,403 | 9,001 |

### 3. Label Schema

The converter normalizes the source IOB tags to these labels:

| Raw tag | Label |
| --- | --- |
| `NEP` | `person` |
| `NEL` | `location` |
| `NEO` | `organization` |
| `NEM` | `measure` |
| `NETI` | `time` |
| `NED` | `date` |
| `ED` | `designation` |

Training labels:

```json
["person", "location", "organization", "measure", "time", "date", "designation"]
```

### 4. Useful Conversion Options

Include source metadata in each example:

```bash
python3 scripts/convert_iob_to_gliner.py --include-provenance
```

Fail on malformed `I-*` tags instead of repairing them as new spans:

```bash
python3 scripts/convert_iob_to_gliner.py --strict-iob
```

Convert a single split or source:

```bash
python3 scripts/convert_iob_to_gliner.py --split train --source l3cube-mahaner
```

## Label Studio Export

Convert GLiNER JSONL into Label Studio import JSON:

```bash
python3 scripts/convert_label_studio.py
```

The current script converts:

```text
data/processed/gliner/valid.jsonl
```

to:

```text
data/processed/gliner/labelstudio/valid.json
```

Label Studio tasks contain:

- `data.text`: whitespace-joined sentence text
- `annotations.result`: character-offset entity spans
- `labels`: one normalized NER label per span

Use this export to inspect the converted dataset, correct annotations manually,
or prepare reviewed examples for future training.

## Train Lightweight GLiNER Model

The current training workflow is in:

```text
Copy_of_GliNER_Finetune.ipynb
```

The notebook uses:

- base checkpoint: `urchade/gliner_multi`
- training data: `data/processed/gliner/train.jsonl`
- evaluation data: processed validation/test JSONL
- labels: `person`, `location`, `organization`, `measure`, `time`, `date`,
  `designation`

Typical notebook flow:

1. Install `gliner[training]`.
2. Load the processed Marathi GLiNER JSONL dataset.
3. Load `GLiNER.from_pretrained("urchade/gliner_multi")`.
4. Fine-tune with `model.train_model(...)`.
5. Save the trained checkpoint.
6. Evaluate on held-out test data.
7. Run sample Marathi inference with the trained model.

## Run Inference Checks

GLiNER smoke test:

```bash
python3 scripts/test_gliner_marathi.py
```

With custom labels:

```bash
python3 scripts/test_gliner_marathi.py \
  --labels person,organization,location,date \
  --threshold 0.15
```

MahaBERT token-classification inference:

```bash
python3 scripts/infer_mahabert_ner.py
```

With a specific model and JSON output:

```bash
python3 scripts/infer_mahabert_ner.py \
  --model l3cube-pune/marathi-ner \
  --json
```
